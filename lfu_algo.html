LFU (Least Frequently Used)

"The LFU cache is a clever algorithm for managing cache memory efficiently"


Least Frequently Used (LFU) is a caching algorithm in which the least frequently used cache block is removed whenever the cache is overflowed. In LFU we check the old page as well as the frequency of that page and if the frequency of the page is larger than the old page we cannot remove it and if all the old pages are having same frequency then take last i.e FIFO method for that and remove that page.

keyboard_double_arrow_right Use Counter:

Each cache item (key-value pair) has an associated use counter. The use counter is initialized to 1 when a new item is inserted into the cache. The use counter is incremented every time a get or put operation is performed on the item.

keyboard_double_arrow_right Least Frequently Used Key:

The core idea of the LFU algorithm is to identify and remove the least frequently used item from the cache when it reaches its capacity. The item with the smallest use counter is considered the least frequently used key.

keyboard_double_arrow_right Time Complexity:

One of the main advantages of the LFU cache algorithm is its constant time complexity (O(1)) for basic operations like get and put. This is achieved by efficiently managing the cache items using the linked list and hashmap data structures.

keyboard_double_arrow_right Efficiency:

LFU is especially effective in scenarios where certain items are accessed frequently while others are accessed infrequently. By evicting the least frequently used items, the cache optimally manages its space to store more relevant data.

keyboard_double_arrow_right Eviction Strategy:

When the cache is full and a new item needs to be inserted, the LFU algorithm removes the item with the smallest use counter. This eviction strategy ensures that the cache retains the most frequently and recently used items.

keyboard_double_arrow_right Adaptiveness:

The LFU cache adapts well to changing usage patterns over time. If an item that was initially accessed infrequently becomes more popular, its use counter will increase, preventing it from being evicted prematurely.
